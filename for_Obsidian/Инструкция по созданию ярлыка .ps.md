# Инструкция по созданию ярлыка для запуска приложения

## Способ 1: Автоматическое создание ярлыка

В проекте есть скрипт PowerShell, который автоматически создаст ярлык на рабочем столе:
1. Откройте PowerShell от имени администратора (если требуется) или обычный PowerShell
2. Перейдите в директорию проекта:
   ```powershell
   cd путь_к_проекту\Hex2BinAndSelectCheck
   ```
3. Выполните скрипт создания ярлыка:
  ```powershell
   .\create_shortcut.ps1
  ```
4. После успешного выполнения вы увидите сообщение о создании ярлыка на рабочем столе
## Способ 2: Ручное создание ярлыка

Если автоматический способ не работает, можно создать ярлык вручную:

1. Щелкните правой кнопкой мыши на рабочем столе
2. Выберите "Создать" → "Ярлык"
3. В поле "Расположение объекта" введите:
  ```
   powershell.exe -NoExit -ExecutionPolicy Bypass -File "полный_путь_к_проекту\Hex2BinAndSelectCheck\run.ps1"
   ```
   Замените "полный_путь_к_проекту" на фактический путь к вашему проекту

4. Нажмите "Далее"
5. Введите имя ярлыка, например "Hex2BinAndSelectCheck"
6. Нажмите "Готово"

## Настройка ярлыка (опционально)

После создания ярлыка можно настроить дополнительные параметры:

1. Щелкните правой кнопкой мыши на созданном ярлыке
2. Выберите "Свойства"
3. На вкладке "Ярлык" можно изменить:
   - "Рабочая папка" - укажите путь к директории проекта
   - "Запускать в" - выберите "Обычное окно", "Развернутое" или "Свернутое"
   - Добавить иконку через кнопку "Сменить значок"
1. Нажмите "ОК", чтобы сохранить изменения
## Примечания

- Ярлык запускает PowerShell с параметром `-NoExit`, чтобы окно консоли оставалось открытым после выполнения скрипта
- Параметр `-ExecutionPolicy Bypass` позволяет запускать скрипты PowerShell без изменения политики выполнения системы
- Если возникают проблемы с запуском ярлыка, проверьте, что пути корректны и у вас есть соответствующие права доступа
## Содержимое файла create_shortcut.ps1

Если вы хотите изучить скрипт создания ярлыка, вот его содержимое:
```powershell
# Скрипт для создания ярлыка на рабочем столе

# Получаем полный путь к текущей директории и файлу run.ps1
$scriptPath = Join-Path -Path $PWD -ChildPath "run.ps1"
# Путь к рабочему столу
$desktopPath = [Environment]::GetFolderPath("Desktop")
# Имя ярлыка
$shortcutName = "Hex2BinAndSelectCheck.lnk" 
# Полный путь к ярлыку
$shortcutPath = Join-Path -Path $desktopPath -ChildPath $shortcutName
# Создаем объект COM WScript.Shell для работы с ярлыками
$shell = New-Object -ComObject WScript.Shell
  
# Создаем ярлык
$shortcut = $shell.CreateShortcut($shortcutPath)
$shortcut.TargetPath = "powershell.exe"
$shortcut.Arguments = "-NoExit -ExecutionPolicy Bypass -File `"$scriptPath`""
$shortcut.WorkingDirectory = $PWD.Path
$shortcut.Description = "Запуск Hex2BinAndSelectCheck"
$shortcut.IconLocation = "powershell.exe,0"
$shortcut.Save()
 
Write-Host "Ярлык создан на рабочем столе: $shortcutPath" -ForegroundColor Green
```
